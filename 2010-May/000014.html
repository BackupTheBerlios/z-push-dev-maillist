the Kolab backend is for Kolab groupware , each groupware has his 
solution or need a backend 
developping  another backend is not our priority.

if you want to have the code this is the right url for the repositionny 
but i dont think that it help you a lot
for a caldav backend.

http://kolabzpush.svn.sourceforge.net/viewvc/kolabzpush/trunk/Kolab_Zpush
but if you want to start a backend you must have somethings :
- a good code for a caldav and groupdav client protocol.
- have an method to see what is changed on the server
and after you must have a class derived from diffBackend
and rewrite the methods who manage the datas
