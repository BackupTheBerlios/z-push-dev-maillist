Hi Christoph,

 

Z-Push releases basically have the following structure: MajorRelease.MinorRelease.VersionCounter

 

Z-Push 2.0 is a new major release because we have changed a lot of things in code, the API has changed, state and device management have changed and there are new features. The 1.5.X versions will only receive bug fixes. After Z-Push 2.0 is stable and released, support for 1.X releases will be discountinued after transition period.

 

The roadmap and release schedule are available at http://z-push.sf.net/roadmap. Currently there is no API available. The goal of early releases is to get the feedback about the functionality as soon as possible in order to have as few bugs in the stable release as possible.
