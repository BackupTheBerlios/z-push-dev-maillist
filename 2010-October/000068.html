Hi Alain,

 

the M4E client is really giving everybody a hard time. It has flaws, much also reported to be present with exchange. If you search for "M4E system error - stop syncing" or similar you will find many relates.

I had cases, that I needed a full hard reset to convince the device to even connect to the server again.

 

I know that this client has major issues with stuff like:

- folderids too long (or not in base64)

- add/change requests on folders which data was already synced (foldercache should handle it)

- missing flags or not only flags when synching messages (update on emails)

 

I think that the issue is not in the actual activesync implementation. I'm quite sure that the M4E client doesn't "like" the data it receives.

 

Greetings,

Sebastian
