Hi everybody,

 

during supporting Philip and Ralf to develop a backend for the groupware they use we discovered a problem in DiffBackend.php file. The function GetDiff calls usort with sorting function RowCmp. In RowCmp it says:

 

return $a["id"] < $b["id"] ? 1 : -1;

 

This produces upon return values that usort expect wrong and unpredictable results. Usort expects either 1, -1 or 0 (0=both values are equal) as a result.

 

To return correct values we’ve changed the function RowCmp the following way and it works by this as expected:

 

return strcmp($a["id"],$b["id"]);

 

Best regards

 

Andreas
